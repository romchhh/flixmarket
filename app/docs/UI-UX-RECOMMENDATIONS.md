# Рекомендації з UI/UX мінідодатку Flix Market

Пропозиції по сторінках: що покращити для зручності, читабельності та конверсії.

---

## Головна (Home / CatalogShell)

| Що покращити | Пояснення |
|--------------|-----------|
| **Ліміт 40 000 ₴** | Зараз статичний текст. Якщо є API/логіка — показувати реальний ліміт користувача; якщо ні — або прибрати блок, або зробити його інфо-карткою («Як працює ліміт» з посиланням у бота). |
| **Баннер «Підписки від 1 місяця»** | Додати одну чітку CTA: «Перейти в каталог» або «Обрати підписку», щоб баннер був клікабельним. |
| **Пошук** | Додати кнопку очищення (×) у праву частину поля, коли є текст. Підсвічувати кількість результатів: «Знайдено 5 товарів». |
| **Категорії** | На головній показуються лише іконки — дубль з каталогу. Можна показувати фото категорій (якщо є `catalog_photo`) замість емодзі для консистентності з каталогом. |
| **Картки товарів** | Додати міні-скілетон (сірі плейсхолдери) під час завантаження замість порожнього екрану. При довгому списку — підтягувати ще при скролі (infinite scroll) або кнопка «Показати ще». |
| **Порожній пошук** | Якщо нічого не знайдено — показати варіанти: «Спробуйте інший запит», «Перейти в каталог», можливо популярні категорії. |

---

## Каталог (список категорій)

| Що покращити | Пояснення |
|--------------|-----------|
| **Єдиність стилю** | Використовувати `catalog_photo` з API, якщо є — як на головній, так і тут, щоб не було розбіжностей (емодзі тут, фото там). |
| **Назви категорій** | Обрізати довгі назви з `line-clamp-2` або показувати тултип при довгій назві. |
| **Завантаження** | Скілетон сітки категорій (наприклад 6 карток) замість одного спінера по центру. |
| **Порожній стан** | Якщо категорій немає — окремий екран з текстом і (за потреби) посиланням у бота. |

---

## Сторінка категорії (товари в категорії)

| Що покращити | Пояснення |
|--------------|-----------|
| **Заголовок** | Показувати назву категорії в хедері (зараз лише «Товари»). Наприклад: «YouTube Premium та інші». |
| **Сортування/фільтри** | Опційно: «За ціною», «За популярністю» (якщо з’явиться метрика). Хоча б один перемикач покращить відчуття контролю. |
| **Консистентність карток** | Однаковий стиль карток з головною та сторінкою товару: однакова висота зображення, однакові відступи, бейджі в одному місці. |
| **Кнопка «Назад»** | Залишити як є; можна додати `router.back()` замість жорсткого `/catalog`, щоб повертатися туди, звідки прийшли (головна чи каталог). |

## Сторінка товару

| Що покращити | Пояснення |
|--------------|-----------|
| **Фото** | Додати можливість відкрити фото на весь екран (lightbox) по тапу. Для Telegram Web App — можливо використати `window.Telegram.WebApp.openLink` або просто збільшити в модалці. |
| **Кнопки тарифів** | На мобільному одна кнопка = один тариф. Можна додати іконку «відкрити в Telegram» (наприклад, іконку Telegram) біля кожної кнопки, щоб було зрозуміло, що оплата в боті. |
| **«Вам може сподобатися»** | Горизонтальний скрол (карусель) замість сітки 2 колонки — більше видно товарів і менше доведеться скролити вниз. Підпис типу «Інші підписки». |
| **Оформлення в Telegram** | Якщо є лише одна кнопка «Оформити в Telegram» — зробити її максимально помітною (основна CTA), можливо з іконкою Telegram. |
| **Опис** | Якщо опис довгий — «Читати далі» з розгортанням тексту, щоб не займати весь екран. |
| **Стани завантаження/помилки** | Скілетон картки товару під час завантаження. При помилці — кнопка «Повторити» поруч із «Назад до каталогу». |

## Обране (Favorites)

| Що покращити | Пояснення |
|--------------|-----------|
| **Порожній стан** | Не лише текст «Немає обраного», а ілюстрація або іконка + «Додайте товари з головної або каталогу» + кнопка «Перейти в каталог». |
| **Консистентність** | Та сама картка, що на головній/каталозі (фото, ціна, бейдж підписки, бейдж «Гаряча пропозиція» тощо). |
| **Видалення** | Підтвердження при видаленні з обраного (наприклад, «Видалити з обраного?») або швидке свайп-видалення з підказкою. |
| **Заголовок** | У шапці можна показати кількість: «Обране (3)». |

## Профіль

| Що покращити | Пояснення |
|--------------|-----------|
| **Аватар** | Якщо Telegram дає фото профілю (через initData/user) — показувати його замість іконки. |
| **Блок «На сервісі N днів»** | Можна зробити візуально окремою міні-карткою або бейджем, щоб виглядало як досягнення. |
| **Підписки** | Якщо підписок багато — секція «Активні» зверху, далі «Закінчились» або згорнутий блок «Історія». |
| **Історія платежів** | За замовчуванням показувати останні 5–10, далі «Показати всі» або окрема сторінка «Історія платежів». Групувати по місяцях (листопад 2025, жовтень 2025). |
| **Навігація** | Ясні посилання: «Мої підписки», «Каталог», можливо «Підтримка» (посилання на бота або чат). |
| **Помилки / не авторизований** | Короткий текст + одна кнопка «Відкрити в Telegram» (deep link у бота), щоб не лишати користувача без дії. |

## Мої підписки

| Що покращити | Пояснення |
|--------------|-----------|
| **Розділення** | Активні зверху з таймером, закінчені — окремим блоком внизу або вкладками «Активні» / «Всі». |
| **Платежі по підписці** | Зараз по кліку показуються платежі — можна додати невеликий індикатор на картці: «3 платежі», щоб було зрозуміло, що є що розгорнути. |
| **Порожній стан** | Текст + CTA «Перейти в каталог» + можливо посилання «Як продовжити підписку» (у бота). |
| **Таймер** | Якщо до кінця менше доби — підсвітити картку (наприклад, жовтий/оранжевий бордер) і текст «Закінчується сьогодні» або «Залишилось X годин». |

## Нижня навігація (BottomNav)

| Що покращити | Пояснення |
|--------------|-----------|
| **Іконка «Підписки»** | Зараз Bell — можна CreditCard або щось, що асоціюється з підписками, щоб відрізняти від сповіщень. |
| **Бейдж на «Обране»** | Показувати кількість обраних (наприклад, кружечок з числом) на іконці «Обране». |
| **Бейдж на «Підписки»** | Якщо є активні підписки — невеликий індикатор (крапка або число), щоб заохотити зайти. |
| **Safe area** | Додати `padding-bottom` з урахуванням `env(safe-area-inset-bottom)` для iPhone з вирізом, щоб навігація не наїжджала на жести. |
| **Підсвітка активного пункту** | Залишити поточну логіку; можна додати невелику анімацію при перемиканні (наприклад, scale або fade). |

## Загальні покращення

| Що покращити | Пояснення |
|--------------|-----------|
| **Єдиний спінер/скілетон** | Один компонент для стану завантаження (наприклад, `PageLoader` або сітка скілетон-карток) і використовувати його на всіх сторінках. |
| **Посилання «Назад»** | Де можливо — `router.back()` замість жорсткого посилання, щоб користувач повертався туди, звідки прийшов. |
| **Телеграм-тематика** | Використати `window.Telegram.WebApp` для кольору хедеру (`setHeaderColor`), кнопки (`MainButton`) на сторінці товару для CTA, закриття Web App після оплати — для більш нативного відчуття. |
| **Доступність** | Для кнопок і посилань додати `aria-label` там, де лише іконка; перевірити контраст тексту (наприклад, сірий на білому). |
| **Помилки мережі** | Якщо fetch падає (немає інтернету) — показувати загальний екран «Щось пішло не так» з кнопкою «Повторити» замість тихої помилки. |
| **Мета та title** | Для кожної сторінки окремий `title` (наприклад, «Товар — Flix Market», «Профіль — Flix Market») для вкладки/історії. |

## Пріоритизація

**Швидкі wins (мало коду, помітний ефект):**
- Кнопка очищення пошуку та «Знайдено N товарів».
- Бейдж кількості на «Обране» в BottomNav.
- Назва категорії в хедері сторінки категорії.
- Покращені порожні стани (Обране, Профіль без даних, Нічого не знайдено).
- Safe area для нижньої навігації.

**Середні (помірний об’єм робіт):**
- Скілетони завантаження замість спінерів.
- Групування платежів по місяцях на профілі.
- Розділ «Активні» / «Закінчені» на «Мої підписки».
- Lightbox для фото товару.
- Карусель «Вам може сподобатися».

**Довгострокові (потрібні бекенд або більше логіки):**
- Реальний ліміт на підписки на головній.
- Фото профілю з Telegram.
- Сортування/фільтри в категорії.
- Використання MainButton/HeaderColor Telegram Web App.

Можу допомогти реалізувати будь-який пункт із цього списку поетапно (наприклад, спочатку пошук і порожні стани, потім профіль і підписки).
